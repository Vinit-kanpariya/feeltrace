### Full E2E Flow — WIRED ✅

```
User → AnalyzeForm (/)
  → POST /api/analyze (ssrf.ts + rate limiter)
  → Job created in Neon DB
  → QStash publishes to RAILWAY_CRAWLER_URL
  → Crawler /crawl (HMAC-verified)
  → processJob: pending→crawling→extracting→analyzing
  → Dual viewport crawl (mobile 375px + desktop 1440px)
  → Signal extraction (DOM, CSS, JS, Network) — in-memory only
  → runAIPipeline(jobId, signals)
    → Stage 1: scoreSignals() → 23 threshold rules → ScoredIssue[]
    → Stage 2: Groq llama-3.3-70b → EnrichedIssue[] + CausalEdgeCandidate[]
    → Stage 3: Groq llama-3.3-70b → NarrativeResult
    → Prisma transaction: Result + Issue[] + CausalEdge[] written to Neon
  → job.status = 'complete'
  → JobStatusBadge (polls every 2s) detects complete → router.push(/results/[jobId])
  → /results/[jobId] Server Component (Prisma direct query)
    → NarrativeSection (summary + perceived + technical + recommendations)
    → IssueCard × N (ordered severity DESC)
    → CausalityGraph (if meetsCredibilityThreshold) or GraphAbsent
  → ShareButton → clipboard copy of /results/[jobId]
```

### Integration Findings by Severity

**WARNINGS (not blockers):**

1. **`getGeminiClient` is dead code** — `crawler/src/lib/gemini.ts` exports a Google Generative AI singleton that is never imported. `@google/generative-ai` remains as a live dead dependency in `crawler/package.json`. Should be removed to reduce Docker image size and eliminate confusion about the active LLM provider.

2. **Failed jobs render 404-style page, not error panel** — `results/[jobId]/page.tsx` calls `notFound()` when no `Result` exists. Since `processor.ts` only updates `job.status = 'failed'` without creating a `Result`, direct URL navigation to a failed job hits not-found.tsx (generic) instead of an inline error message. The `JobStatusBadge` home-page path is unaffected.

3. **`TechProfile` type contract inconsistency** — `database`, `auth`, `payments`, `services` are required in `crawler/src/lib/types.ts` but optional in `src/types/tech.ts`. `TechStackSection` guards with `?? []` so runtime is safe, but TypeScript permits constructing invalid objects on the Next.js side.

4. **`RAILWAY_CRAWLER_URL` sync risk** — used as QStash publish target and as QStash signature verification URL. A mismatch between the two services causes silent 401s with no alerting. Crawler startup env var validation would close this.

5. **Screenshot proxy uses raw `fetch` against private Vercel Blob** — works at runtime but bypasses `@vercel/blob` SDK; fragile if Vercel changes private Blob auth semantics.
