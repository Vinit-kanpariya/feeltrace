model Result {
  id         String       @id @default(cuid())
  jobId      String       @unique
  narrative  Json         // stores NarrativeResult object
  issues     Issue[]
  edges      CausalEdge[]
}

model Issue {
  category              String
  signal_source         String
  severity              Int        // Critical=4, High=3, Medium=2, Low=1
  raw_evidence          String
  technical_description String     // NON-NULLABLE
}

model CausalEdge {
  relationship String
  confidence   String
  mechanism    String  // NON-NULLABLE — Prisma rejects create without this
  explanation  String  // NON-NULLABLE — use ?? '' fallback
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| enriched issues → Anthropic API (Stage 3) | JSON of typed objects — no raw HTML or user-supplied strings |
| Anthropic API response → DB | Narrative text is stored as a Json field in Result; not executed or rendered without escaping |
| run-pipeline → Prisma | All writes go through Prisma ORM parameter binding — no raw SQL interpolation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Tampering | LLM narrative containing injected instructions | accept | Narrative is stored as Json and rendered as plain text in Phase 4 — no eval or innerHTML; stored text cannot execute |
| T-03-11 | Denial of Service | LLM cost overrun on Stage 3 | mitigate | max_tokens: 1024 cap per Stage 3 call; two-call total per job bounded by INFRA-02 upstream rate limit |
| T-03-12 | Tampering | CausalEdge mechanism null constraint violation at DB write | mitigate | Stage 2 zod validates mechanism before reaching run-pipeline; explanation uses ?? '' fallback |
| T-03-13 | Tampering | Partial DB write (Result created but CausalEdges fail) | mitigate | prisma.$transaction wraps all writes — failure in causalEdge.createMany rolls back the entire transaction including the Result |
</threat_model>
