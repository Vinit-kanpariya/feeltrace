## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, severity dot offset |
| sm | 8px | Inline label gaps, compact list item padding |
| md | 16px | Card internal padding, form element spacing |
| lg | 24px | Section padding, between issue cards |
| xl | 32px | Between major sections (narrative, issue list, graph) |
| 2xl | 48px | Top-of-page above narrative section |
| 3xl | 64px | Page bottom margin |

Exceptions:
- Touch target minimum: 44px for the copy-to-clipboard button (WCAG 2.5.5 AA)
- React Flow canvas: height 480px on desktop, 320px on mobile (fixed pixel, not spacing scale)

## Interaction Contracts

### Causality Graph Credibility Threshold

The graph renders only when ALL of the following are true:
1. The Result record has at least 1 `CausalEdge` record.
2. At least 1 CausalEdge has `confidence = 'high'`.
3. The total edge count is >= 2.

If any condition fails, `GraphAbsent` renders instead. The graph section heading ("Why these issues are connected") is also absent — no heading orphan.

Rationale: STATE.md note — "Causality graph is the highest trust risk — cap MVP at 3-5 high-confidence edges." Rendering a single low-confidence edge would undermine user trust in the analysis.

### Share Link Copy

1. User clicks "Copy share link".
2. `navigator.clipboard.writeText(window.location.href)` is called.
3. On success: button enters success state (icon + label change, fill inverts) for exactly 2000ms, then reverts to rest state.
4. On failure (clipboard API denied or unavailable): `alert("Could not copy link. URL: " + window.location.href)` — no silent failure.
5. No server call is made. The share link is the current page URL; it is already persistent because `Result` records are not TTL'd in Phase 4.

### Severity Badge Display

- Severity integer from `Issue.severity` maps to label via `SEVERITY_LABELS` from `crawler/src/pipeline/types.ts`.
- Color is determined by the severity color mapping table in the Color section above.
- No runtime label override is permitted — label is always "Critical", "High", "Medium", or "Low".
- Issues with `severity = 4` (Critical) receive additional treatment: the IssueCard left-border is 4px solid `#dc2626` and the card appears at the top of the list.

### Progress State (Post-Job Completion Navigation)

When `JobStatusBadge` (existing component in `src/components/`) detects `status === 'complete'`, it navigates to `/results/{jobId}` using `router.push`. This is a cross-component handoff — `JobStatusBadge` routes, `ResultsPage` renders.

### Loading State (ResultsPage)

`ResultsPage` is a Server Component. Next.js App Router `loading.tsx` at the results route segment provides the loading skeleton:
- Narrative section: 3 gray shimmer lines at body height.
- Issue list: 3 skeleton cards, each with a badge-width shimmer + 2 text-line shimmers.
- Graph panel: Single gray rectangle, graph dimensions, labeled "Loading analysis...".

### Recommendations List

`NarrativeResult.recommendations` is a `string[]`. Render as an unordered list (`<ul>`) with 8px (sm) gap between items. Each item prefixed with a Lucide `ArrowRight` icon (14px, accent color) inline-start.
