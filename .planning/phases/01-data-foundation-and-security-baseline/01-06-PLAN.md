<objective>
Implement Vercel Edge Middleware at middleware.ts with two independent sliding window rate limiters: 5/hour and 20/day per IP (D-06). The middleware runs before POST /api/analyze to prevent mass job submission and uncontrolled API spend.

Purpose: Rate limiting must be at the Edge Middleware layer — it runs before the API route handler, preventing even the SSRF check and DB query from executing when a limit is exceeded. This is the cheapest defense against abuse.

Output: middleware.ts at the project root, containing two Ratelimit instances (one hourly, one daily) with independent prefix keys, returning 429 with Retry-After header on limit exceeded.
</objective>

    Create middleware.ts at project root (not inside src/) per RESEARCH.md Pattern 5 exactly. Key points:

    Imports:
    - import { NextFetchEvent, NextRequest, NextResponse } from 'next/server'
    - import { Ratelimit } from '@upstash/ratelimit'
    - import { redis } from '@/lib/redis' (shares the Redis singleton — D-07, keeps counters in the dedicated Redis database)

    Two SEPARATE Ratelimit instances (CRITICAL — one instance cannot enforce two independent time windows):
    - hourlyLimiter: new Ratelimit({ redis, limiter: Ratelimit.slidingWindow(5, '1 h'), prefix: 'rl:hourly' })
    - dailyLimiter: new Ratelimit({ redis, limiter: Ratelimit.slidingWindow(20, '1 d'), prefix: 'rl:daily' })

    Middleware function:
    - const ip = request.ip ?? '127.0.0.1' (Pitfall 3 — undefined in local dev)
    - Check hourly limit first: const hourly = await hourlyLimiter.limit(ip)
    - If !hourly.success: const retryAfter = Math.ceil((hourly.reset - Date.now()) / 1000); return new Response('Too many requests. Try again in {X} minutes.', { status: 429, headers: { 'Retry-After': String(retryAfter), 'Content-Type': 'text/plain' } })
    - Check daily limit: const daily = await dailyLimiter.limit(ip)
    - If !daily.success: similar response with hours instead of minutes
    - return NextResponse.next()

    Matcher config (CRITICAL — only rate limit /api/analyze, not all routes):
    - export const config = { matcher: '/api/analyze' }

    IMPORTANT: middleware.ts runs in the Vercel Edge Runtime — it cannot import Node.js built-ins (no 'node:dns', no 'node:crypto'). The redis import from @upstash/redis is edge-compatible (HTTP-based). The Ratelimit import from @upstash/ratelimit is edge-compatible.
