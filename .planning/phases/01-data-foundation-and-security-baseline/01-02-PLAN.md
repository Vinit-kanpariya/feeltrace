    Test 4: validateUrl('https://internal.example.com') with lookup returning [{ address: '192.168.1.1', family: 4 }] → rejects with SsrfError { code: 'BLOCKED_IP' }
    Test 5: validateUrl('http://localhost') with lookup returning [{ address: '127.0.0.1', family: 4 }] → rejects with SsrfError { code: 'BLOCKED_IP' }
    Test 6: validateUrl('http://metadata.example.com') with lookup returning [{ address: '169.254.169.254', family: 4 }] → rejects with SsrfError { code: 'BLOCKED_IP' }
    Test 7: validateUrl('file:///etc/passwd') → rejects with SsrfError { code: 'BLOCKED_SCHEME' } (no DNS lookup needed)
    Test 8: validateUrl('ftp://example.com/file') → rejects with SsrfError { code: 'BLOCKED_SCHEME' } (no DNS lookup needed)
    Test 9: validateUrl('http://example.com') with lookup returning [{ address: '::1', family: 6 }] → rejects with SsrfError { code: 'BLOCKED_IP' }
  </behavior>
  <implementation>
    Implement validateUrl per RESEARCH.md Pattern 3 (SSRF Validator section, verbatim). Key points:
    - Export class SsrfError extends Error with code property typed as union: 'BLOCKED_SCHEME' | 'INVALID_URL' | 'DNS_RESOLUTION_FAILED' | 'BLOCKED_IP'
    - Export async function validateUrl(rawUrl: string): Promise<void>
    - Parse URL with new URL(rawUrl); throw SsrfError('Invalid URL format', 'INVALID_URL') on parse failure
    - Check protocol: allow only 'http:' and 'https:'; throw SsrfError with 'BLOCKED_SCHEME' otherwise (per D-14)
    - Call dns.promises.lookup(hostname, { all: true }) — { all: true } is mandatory to get ALL resolved addresses
    - For each resolved address:
      - family === 4: split by '.', map to Number[], run through BLOCKED_RANGES array (loopback 127.x, RFC-1918 10.x / 172.16–31.x / 192.168.x, link-local 169.254.x, unspecified 0.0.0.0)
      - family === 6: block '::1' (loopback) and addresses starting with 'fe80:' (link-local)
    - Import: import dns from 'node:dns'
    - Do NOT import from '@prisma/client' — this file has no DB dependency
  </implementation>
</feature>
