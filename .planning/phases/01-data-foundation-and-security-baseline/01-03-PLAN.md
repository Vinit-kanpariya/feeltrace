<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| src/lib/prisma.ts → Neon PostgreSQL | DATABASE_URL contains DB credentials; never exposed to client; server-side only |
| src/lib/qstash.ts → Upstash QStash | QSTASH_TOKEN is a server-side secret; NEXT_PUBLIC_ prefix must never be used |
| src/lib/redis.ts → Upstash Redis | Redis token is server-side only; used by Edge Middleware (also server-side) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Information Disclosure | Database credentials in process.env.DATABASE_URL | mitigate | Accessed only in src/lib/prisma.ts (server-side); never in any "use client" component; no NEXT_PUBLIC_ prefix |
| T-01-07 | Information Disclosure | QStash token in process.env.QSTASH_TOKEN | mitigate | Accessed only in src/lib/qstash.ts (server-side); never client-facing |
| T-01-08 | Spoofing | QStash callback without signature verification | mitigate | verifySignatureAppRouter with QSTASH_CURRENT_SIGNING_KEY + QSTASH_NEXT_SIGNING_KEY — implemented in Plan 05 crawler callback route |
| T-01-09 | Tampering | CausalEdge created without mechanism field | mitigate | mechanism is non-nullable String in schema (D-17); Prisma will reject any create without it at the DB level |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No new installs; all deps from Plan 01 |
</threat_model>
