<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /api/analyze | Untrusted URL string enters here; zod validates shape, SSRF validates content |
| POST /api/analyze → Neon | Job created only after SSRF check passes and queue depth is within limit |
| POST /api/analyze → QStash | Job ID and URL published to external queue; RAILWAY_CRAWLER_URL must be an HTTPS endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12 | Tampering | POST /api/analyze — SSRF via submitted URL | mitigate | validateUrl from @/lib/ssrf resolves hostname via dns.promises.lookup { all: true } + RFC-1918 integer range check before any DB write or QStash publish |
| T-01-13 | Denial of Service | POST /api/analyze — mass job submission | mitigate | Edge Middleware rate limit (Plan 06) blocks most abuse; queue depth cap of 50 (D-09) is second line of defense inside the route |
| T-01-14 | Information Disclosure | Error responses exposing stack traces | mitigate | All catch blocks return only { error: string, code?: string } — no exception.message propagation, no internal paths |
| T-01-15 | Denial of Service | QStash publish failure leaving job in limbo | mitigate | On publishJSON failure: job is updated to status 'failed' with error_message; client polling will see 'failed' and stop |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No new installs in this plan |
</threat_model>
