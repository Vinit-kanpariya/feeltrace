}

         - Build HAREntry and push to entries array

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HAR file (disk) → Node.js JSON.parse | HAR content comes from Playwright's own recording — trusted, but parsing untrusted HAR (e.g., if temp file is replaced) could cause issues |
| page.evaluate() return → JSSignals | JS evaluation result is deserialized from browser context into Node.js |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Tampering | primaryCtaText / frameworkFingerprint strings from page content | mitigate | frameworkFingerprint is a whitelist (`['nextjs', 'react', 'vue3', ...]`) — arbitrary strings from window properties are never included; primaryCtaText is truncated to 100 chars in dom.ts |
| T-02-13 | Information Disclosure | HAR file left on disk after parsing | mitigate | `fs.unlink(harPath).catch(() => {})` in extractNetworkSignals cleans up; HAR has `content: 'omit'` (no response bodies) |
| T-02-14 | Denial of Service | Maliciously large HAR (attacker loads 10K resources) | accept | HAR entries are iterated without storage of raw response bodies (content: 'omit'); NetworkSignals.entries array is bounded by the number of network requests (Playwright captures real requests only) |
| T-02-SC | Tampering | No new npm installs in this plan | accept | All packages installed in 02-01; no additional packages required for these extractors |
</threat_model>
