<objective>
Build the Hono HTTP server that receives QStash job callbacks, verifies signatures, and enqueues jobs for sequential processing. This plan covers the network boundary of the crawler service — everything from the TCP socket to the job queue. No crawl logic runs here.

Purpose: QStash needs a live endpoint to deliver jobs to. Without this, no crawl can be triggered. The signature verification pattern (raw body read before parsing) is the critical correctness constraint — get it wrong and every delivery fails with 401.

Output: A Hono server at `crawler/src/server.ts` with `/health` and `/crawl` routes, a p-queue singleton at `crawler/src/queue.ts`, and an updated `crawler/src/index.ts` entry point that starts the server.
</objective>

    Create `crawler/src/queue.ts`:
    - p-queue v9 is ESM-only. In a CommonJS project (`"type": "commonjs"` in package.json), dynamic import is required at module initialization.
    - Pattern: use a module-level promise that resolves the queue instance, or initialize synchronously if TypeScript allows. The safest CommonJS-compatible pattern is:

    ```typescript
    // crawler/src/queue.ts
    // p-queue v9 is ESM-only — load via dynamic import in a CommonJS context
    import type PQueueClass from 'p-queue'

    // Resolved queue instance — initialized in initQueue(), called before first use
    let _queue: InstanceType<typeof PQueueClass> | null = null

    export async function getQueue(): Promise<InstanceType<typeof PQueueClass>> {
      if (_queue) return _queue
      const { default: PQueue } = await import('p-queue')
      _queue = new PQueue({ concurrency: 1 })  // D-23: one job at a time
      return _queue
    }

    // Synchronous accessor — only safe after initQueue() has resolved
    export function queue(): InstanceType<typeof PQueueClass> {
      if (!_queue) throw new Error('[queue] Not initialized — call initQueue() before use')
      return _queue
    }

    export async function initQueue(): Promise<void> {
      await getQueue()
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| QStash → Railway /crawl | Untrusted HTTP POST — any actor on the internet can attempt to POST to this endpoint |
| rawBody → JSON.parse | After signature verification, body is trusted; zod validation prevents malformed shapes from reaching the queue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Spoofing | POST /crawl — unauthenticated job injection | mitigate | QStash `Receiver.verify()` verifies HMAC JWT signature on raw body; 401 returned for missing or invalid signature (D-03) |
| T-02-05 | Tampering | QStash signature verified on re-serialized JSON | mitigate | `c.req.text()` reads raw body before any parsing; signature verified against raw bytes, not `JSON.stringify(parsed)` (RESEARCH.md Pitfall 3) |
| T-02-06 | Denial of Service | Unbounded job queue memory growth | mitigate | p-queue with concurrency:1 processes sequentially; QStash rate-limits delivery to 500/day on free tier; no queue size cap needed for MVP |
| T-02-07 | Elevation of Privilege | RAILWAY_PUBLIC_URL misconfiguration in Receiver.verify | mitigate | URL must exactly match the Railway service public URL; mismatch causes all deliveries to fail with 401 — documented in env var setup |
| T-02-SC | Tampering | npm install of @hono/node-server if added | mitigate | slopcheck before install; @hono/node-server is from the official Hono organization (honojs) |
</threat_model>
