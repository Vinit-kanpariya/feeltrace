  <action>
    Create `crawler/src/browser.ts` implementing the full dual viewport crawl sequence. Follow RESEARCH.md Patterns 3 and 4 exactly.

    Implement `waitForSpaHydration(page, timeoutMs = 10_000)` from RESEARCH.md Pattern 3:
    1. `await page.waitForLoadState('domcontentloaded')` — waits for DOM to be parsed
    2. `await page.waitForFunction(...)` checking: `window.__NEXT_DATA__` (Next.js), React root fiber (`__reactFiber*` or `__reactContainer*` on `#root` or `#__next`), `window.__vue_app__` (Vue 3), `window.__nuxt` (Nuxt), `document.readyState === 'complete'` (fallback)
    3. Catch timeout — log `[hydration] waitForFunction timed out — proceeding` and continue
    4. `await page.waitForTimeout(500)` — brief pause for late-initializing components

    NEVER use `waitForLoadState('networkidle')` — this is the networkidle trap (RESEARCH.md Pitfall 2, D-25).

    Implement `isPrivateHost(hostname: string): boolean` helper for SSRF Layer 2 (D-13):
    - Returns true for: 10.x.x.x, 172.16-31.x.x, 192.168.x.x, 127.x.x.x, 169.254.x.x, "localhost", "::1"
    - Use string prefix checks (fast, no regex needed for these ranges)

    Implement `crawlWithViewport(browser, url, options)` from RESEARCH.md Pattern 4:
    1. `browser.newContext({ viewport, isMobile, hasTouch, recordHar: { path: harPath, content: 'omit', mode: 'full' } })` — HAR recording at context creation (D-26)
    2. `context.newPage()`, then SSRF Layer 2 route interception via `context.route('**', ...)` calling `isPrivateHost` → `route.abort('blockedbyclient')` for private hosts (D-13)
    3. If options.throttle: `const client = await context.newCDPSession(page)` → `client.send('Network.enable')` → `client.send('Network.emulateNetworkConditions', { offline: false, downloadThroughput, uploadThroughput, latency })` (D-24)
    4. Start coverage: `page.coverage.startCSSCoverage({ resetOnNavigation: false })` and `page.coverage.startJSCoverage({ resetOnNavigation: false, reportAnonymousScripts: false })`
    5. `page.goto(url, { timeout: 40_000, waitUntil: 'domcontentloaded' })` — 40s navigation budget (part of 55s SLA, D-28)
    6. `waitForSpaHydration(page, 10_000)`
    7. Extract signals: call `extractDOMSignals(page)`, `extractCSSSignals(page)` (which internally calls `stopCSSCoverage`)
    8. Stop JS coverage: `page.coverage.stopJSCoverage()` — call before context.close() (Pitfall 4)
    9. `await context.close()` — flushes HAR to disk (Pitfall 4)
    10. Return partial CrawlPass with domSignals and cssSignals populated; jsSignals and networkSignals as typed stubs (`{} as JSSignals` and `{ entries: [] } as NetworkSignals`). Do NOT import or call extractJSSignals/extractNetworkSignals here — those files are created in 02-04 (parallel wave) and wired in 02-05. The stubs allow `npx tsc --noEmit` to pass in this plan's verify step independently without 02-04 files existing.

    The `harPath` must be unique per pass: `/tmp/feeltrace-${jobId ?? Date.now()}-${viewport}.har`

    Implement `runDualViewportCrawl(url: string): Promise<{ mobile: CrawlPass, desktop: CrawlPass }>`:
    1. `chromium.launch({ args: ['--disable-dev-shm-usage', '--disable-gpu'] })` (Pitfall 7 — no `--no-sandbox`)
    2. Pass 1: mobile (375×812, isMobile: true, hasTouch: true, throttle: D-27 values)
    3. Pass 2: desktop (1440×900, isMobile: false, hasTouch: false, throttle: null)
    4. `finally { await browser.close() }` — always close even on error

    Mobile throttle values (D-27): `{ downloadThroughput: 40 * 1024, uploadThroughput: 20 * 1024, latency: 400 }`

    For the HAR path, import `fs/promises` to read after context.close(). Pass harPath to the JS/network extractors (02-04 wires this in).

    Export: `runDualViewportCrawl`, `waitForSpaHydration`, `isPrivateHost` (for testing).

    Create `crawler/src/extractors/css.ts` from RESEARCH.md Pattern 5b.

    Import `Page` from `playwright-core` and `CSSSignals` from `../lib/types`.

    Implement `extractCSSSignals(page: Page): Promise<CSSSignals>`:

    1. `const coverageEntries = await page.coverage.stopCSSCoverage()` — IMPORTANT: this BOTH stops coverage AND returns data. It must be called before `context.close()` (Pitfall 4). The `browser.ts` `crawlWithViewport` function calls `extractCSSSignals` before closing the context for this reason.

    2. Compute `totalCSSBytes` and `unusedCSSBytes` by iterating `coverageEntries`:
       - `totalBytes += entry.text?.length ?? 0`
       - `usedBytes += range.end - range.start` for each range in `entry.ranges`
       - `unusedCSSBytes = totalBytes - usedBytes`
       - `unusedCSSPercent = totalBytes > 0 ? Math.round(((totalBytes - usedBytes) / totalBytes) * 100) : 0`

    3. `const inPageSignals = await page.evaluate(...)` for computed style analysis:
       - Iterate `document.querySelectorAll('*')` with `window.getComputedStyle(el)`
       - `animationCount`: `style.animationName !== 'none'`
       - `transitionCount`: `style.transition !== 'none' && style.transition !== 'all 0s ease 0s'`
       - `willChangeCount`: `style.willChange !== 'auto'`
       - `paintTriggerPropertyCount`: elements with animation OR willChange AND (transform !== 'none' OR opacity < 1 OR filter !== 'none')
       - `fontDisplayMap`: iterate `document.styleSheets` → check `CSSFontFaceRule` → `rule.style.getPropertyValue('font-display')`. Wrap in try/catch for cross-origin SecurityError.

    4. Return assembled `CSSSignals` object.

    Document the CSS-in-JS limitation per RESEARCH.md Pitfall 6: add a JSDoc comment noting that `page.coverage.startCSSCoverage()` does not capture styles injected by JavaScript (CSS-in-JS, Tailwind JIT), so `unusedCSSPercent` is a lower bound, not an exact figure.

    No unit tests for the CSS extractor — Coverage API requires a live Chromium instance. Covered by the end-to-end checkpoint in 02-06.

| page.evaluate() → Node.js | Return values from browser evaluation are deserialized into Node.js — must be typed/validated |

| T-02-09 | Tampering | Prompt injection via page content in DOM signals | mitigate | DOM signals are extracted as typed numbers and counts (never raw HTML strings); primaryCtaText is truncated to 100 chars and is a display hint only — not passed to LLM without sanitization in Phase 3 |
