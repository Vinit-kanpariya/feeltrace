# Plan 06-02: axe-core Accessibility Integration

**Phase:** 06 — Signal Expansion
**Plan:** 02 of 3
**Requirements:** SIGNAL-02
**Goal:** Install `@axe-core/playwright`, run an axe WCAG 2.1 scan on the desktop Playwright pass after SPA hydration, capture violations with selectors and impact levels, and score them into `ScoredIssue[]` via `scoreAxeViolations()`.

---

## Context

This plan delivers SIGNAL-02. `@axe-core/playwright` (Deque's official package) is injected into the desktop browser context after `waitForSpaHydration()` returns. The scan result populates `CrawlPass.axeViolations` (type added in Plan 06-01). `scoreAxeViolations()` in `stage1-external-scorer.ts` (stubbed in Plan 06-01) is filled in with the dedup and cap logic from the research notes. The mobile pass is never scanned — axe runs desktop-only per RESEARCH.md. `bypassCSP: true` is set on the desktop browser context so axe's injected script is not blocked by Content Security Policy headers.

**Prerequisite:** Plan 06-01 must be complete — `AxeViolation`, `AxeViolationNode` types and the `axeViolations?` field on `CrawlPass` must already exist.

---

## Must-Haves

1. `@axe-core/playwright` is listed in `crawler/package.json` dependencies and installed via `sfw npm install`.
2. The desktop browser context in `crawler/src/browser.ts` is created with `bypassCSP: true` — the mobile context is unchanged.
3. After `waitForSpaHydration()` on the desktop pass, an `AxeBuilder` scan scoped to `['wcag2a', 'wcag21aa']` runs and its violations are stored as `AxeViolation[]` on the returned `CrawlPass`.
4. Axe scan failures (exceptions) are caught and logged; the crawl continues and `axeViolations` is set to `[]` on failure rather than crashing the job.
5. `scoreAxeViolations` in `crawler/src/pipeline/stage1-external-scorer.ts` replaces the stub with the real implementation: maps `critical→4`, `serious→3`, `moderate→2`, `minor→1`; caps at 10 unique violation IDs; includes first 5 nodes per violation in `raw_evidence`; emits one `ScoredIssue` per unique violation ID with `viewport: 'desktop'`.
6. `crawler/src/pipeline/run-pipeline.ts` calls `scoreAxeViolations(signals.desktop.axeViolations ?? [])` and concatenates results onto `scoredIssues`.
7. Unit tests in `crawler/src/pipeline/stage1-external-scorer.test.ts` cover the axe scorer: impact-to-severity mapping, 10-ID cap, 5-nodes-per-violation cap, empty input, single violation with multiple nodes.

### Wave 1: Browser Integration

- [ ] **Update `crawler/src/browser.ts`** — desktop context only:
  - Import `AxeBuilder` from `@axe-core/playwright` at the top of the file.
  - In `crawlWithViewport`, when `options.viewport === 'desktop'`, add `bypassCSP: true` to the `browser.newContext(...)` options object. The mobile context call does not receive this option.
  - After `waitForSpaHydration(page, 10_000)` completes (and before the screenshot + fingerprint block), add the axe scan for the desktop pass:
    ```
    if (options.viewport === 'desktop') {
      try {
        const axeResults = await new AxeBuilder({ page })
          .withTags(['wcag2a', 'wcag21aa'])  // WCAG 2.1 A + AA — matches SC-2 requirement
          .analyze()
        axeViolations = axeResults.violations.map((v) => ({
          id: v.id,
          impact: (v.impact ?? 'minor') as AxeViolation['impact'],
          description: v.description,
          helpUrl: v.helpUrl,
          nodes: v.nodes.slice(0, 5).map((n) => ({
            target: n.target.join(', '),
            failureSummary: n.failureSummary ?? '',
          })),
        }))
      } catch (axeErr) {
        console.warn('[browser] axe scan failed:', axeErr instanceof Error ? axeErr.message : axeErr)
        axeViolations = []
      }
    }
    ```
  - Declare `let axeViolations: AxeViolation[] | undefined` before the `if (options.viewport === 'desktop')` screenshot block.
  - Include `axeViolations` on the returned `CrawlPass` object (the `axeViolations?` field added in Plan 06-01).
  - Import `AxeViolation` from `./lib/types` for the type cast.

### Wave 2: Scorer Implementation + Pipeline Wiring

- [ ] **Fill in `scoreAxeViolations` in `crawler/src/pipeline/stage1-external-scorer.ts`** — replace the stub body with real logic:
  - Impact-to-severity map: `critical → 4`, `serious → 3`, `moderate → 2`, `minor → 1`.
  - Collect unique violation IDs only. If two violations share the same `id`, take the first occurrence. Cap the resulting set at 10 IDs (`slice(0, 10)`).
  - For each capped violation, build `raw_evidence` as: `"<id>: <impact> impact — <count> node(s) affected: <target1>; <target2>; ..."` using the first 5 nodes from `violation.nodes` (already pre-sliced by `browser.ts`, but re-slice to 5 defensively).
  - Return one `ScoredIssue` per unique violation with:
    - `category: 'accessibility'`
    - `signal_source: 'axe.<violation.id>'`
    - `severity: <mapped value>`
    - `viewport: 'desktop'` (axe runs on desktop pass only; viewport records where the signal was captured)

- [ ] **Update `crawler/src/pipeline/run-pipeline.ts`** — append axe violations to scored issues:
  - Import `scoreAxeViolations` from `./stage1-external-scorer` (already imported in Plan 06-01; just add to the existing import statement if needed).
  - After the `scoreExternalSignals` append line added in Plan 06-01, add:
    ```
    scoredIssues.push(...scoreAxeViolations(signals.desktop.axeViolations ?? []))
    ```

## Notes

- **Desktop-only constraint:** axe runs only when `options.viewport === 'desktop'`. Never add it to the mobile pass — the mobile `crawlWithViewport` call must not change.
- **`bypassCSP` placement:** The `bypassCSP: true` option belongs on `browser.newContext(...)`, not on a per-page level. Ensure it is inside the conditional block for desktop or use a per-viewport options merge to avoid accidentally setting it on mobile.
- **`v.impact ?? 'minor'` fallback:** axe types declare `impact` as `string | null`. Cast the null case to `'minor'` so the TypeScript union stays satisfied.
- **`n.target` is `string[]`:** `AxeBuilder` node targets are arrays of CSS selector strings. Join them with `', '` to produce a single string for `AxeViolationNode.target`.
- **`n.failureSummary` may be null:** Use `?? ''` fallback when assigning to `AxeViolationNode.failureSummary`.
- **No new test file:** axe scorer tests are added to the existing `stage1-external-scorer.test.ts` created in Plan 06-01.
- **`sfw` prefix:** The package install MUST use `sfw npm install @axe-core/playwright`, not bare `npm install`. This is a project-wide security requirement.
