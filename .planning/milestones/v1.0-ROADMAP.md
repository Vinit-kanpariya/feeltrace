# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-27
**Phases:** 1–4
**Total Plans:** 26

## Overview

FeelTrace v1.0 MVP delivers the full URL → crawl → AI pipeline → results dashboard flow. Users paste a URL, FeelTrace crawls it in dual viewport (mobile + desktop), runs a 3-stage AI pipeline (deterministic scorer → LLM causal reasoner → LLM narrator), and renders a plain-English results page with ranked issues, a causality graph, and a shareable link — no account required.

## Milestone Summary

**Key Decisions:**

| Decision | Rationale |
|----------|-----------|
| Playwright for crawling | SPAs need JS execution; handles React/Vue/Next.js hydrated DOM |
| Structured 3-stage pipeline | Debuggable, cost-efficient; each stage optimized independently |
| Fly.io over Railway | Railway requires payment method; Fly.io identical Docker workflow |
| Groq over Gemini | Gemini free-tier quota exhausted; Groq 14,400 RPD free, no billing |
| QStash as async bridge | Crawl duration 15-45s exceeds synchronous API route safety window |
| No auth in MVP | Shareable-link model validates core AI quality fastest |
| Prisma 7 adapter pattern | url removed from schema datasource (P1012); adapter required |
| In-memory signals only | INFRA-03: raw signal payloads are ephemeral, never stored |

**Issues Resolved:**

- Gemini free-tier quota exhausted across all models — migrated to Groq
- Causal edge index remap bug (LLM indices into scored array ≠ enriched array indices)
- Prisma 7 removed url from schema datasource — PrismaNeon adapter required
- postbuild cpSync needed for Prisma .js/.wasm runtime files (tsc doesn't copy them)
- QStash signature verify URL mismatch — RAILWAY_CRAWLER_URL reused for exact match

**Issues Deferred (tech debt for v1.1+):**

- No VERIFICATION.md files for any phase (procedural gap)
- `getGeminiClient` dead code + dead `@google/generative-ai` dependency in crawler
- Failed jobs render generic 404 instead of inline error on results page
- TechProfile type mismatch across packages
- RAILWAY_CRAWLER_URL cross-service sync risk (no startup validation)
- Screenshot proxy bypasses @vercel/blob SDK

**Technical Debt Incurred:**

- VALIDATION.md files (Nyquist) stuck in draft state — created pre-execution, never updated post-execution
- REQUIREMENTS.md traceability table never updated from "Pending" during execution
